Introduction

Addition and subtraction are fundamental operations in mathematics. While we can perform these operations using standard algorithms, understanding partitioning and place value gives us deeper insight into how numbers work and makes calculations easier.

In this lesson, you will learn:

  • What partitioning means in mathematics
  • How to break numbers into their place value components
  • Methods for adding and subtracting whole numbers using partitioning
  • Techniques for working with decimal numbers
  • Common mistakes to avoid

Real-World Connection

Partitioning is like breaking money into different denominations. If you have GH₵23.50, you can think of it as:

2 × GH₵10 + 3 × GH₵1 + 5 × GH₵0.10

What is Partitioning?

Understanding Partitioning

Partitioning means breaking a number into smaller parts based on its place value. This is also called writing a number in expanded form.

Example 1: Partitioning Whole Numbers

The number 4,567 can be partitioned as:

\[ 4,567 = 4,000 + 500 + 60 + 7 \]

Or using place value:

\[ (4 \times 1,000) + (5 \times 100) + (6 \times 10) + (7 \times 1) \]

Example 2: Partitioning Decimal Numbers

The number 23.45 can be partitioned as:

\[ 23.45 = 20 + 3 + 0.4 + 0.05 \]

Or using place value:

\[ (2 \times 10) + (3 \times 1) + (4 \times 0.1) + (5 \times 0.01) \]

Why Use Partitioning?

Partitioning helps us understand the value of each digit in a number and makes addition/subtraction easier:

Better Understanding

You see what each digit actually represents

Easier Calculations

Breaking problems into simpler steps

Mental Math

Develops strong number sense

Error Detection

Easier to spot mistakes in calculations

Adding & Subtracting Whole Numbers

Addition Using Partitioning

Step-by-Step Method:

  1. Write both numbers in expanded form
  2. Add the components from each place value separately
  3. Combine the results
  4. Adjust for any regrouping needed

Example: Add 347 + 265

Step 1: Partition both numbers

\[ 347 = 300 + 40 + 7 \] \[ 265 = 200 + 60 + 5 \]

Step 2: Add each place value

\[ 300 + 200 = 500 \] \[ 40 + 60 = 100 \] \[ 7 + 5 = 12 \]

Step 3: Combine the results

\[ 500 + 100 + 12 = 612 \]

Final Answer: 347 + 265 = 612

Subtraction Using Partitioning

Step-by-Step Method:

  1. Write both numbers in expanded form
  2. Subtract the components from each place value
  3. If needed, adjust by regrouping (borrowing)
  4. Combine the results

Example: Subtract 532 - 287

Step 1: Partition both numbers

\[ 532 = 500 + 30 + 2 \] \[ 287 = 200 + 80 + 7 \]

Step 2: Subtract each place value (with regrouping)

First subtract hundreds:

\[ 500 - 200 = 300 \]

For tens, we need to adjust because 30 < 80:

\[ 300 = 200 + 100 \] (regroup 1 hundred as 10 tens) \[ \text{Now tens: } (100 + 30) - 80 = 130 - 80 = 50 \]

For ones, we need to adjust because 2 < 7:

\[ 50 = 40 + 10 \] (regroup 1 ten as 10 ones) \[ \text{Now ones: } (10 + 2) - 7 = 12 - 7 = 5 \]

Step 3: Combine the results

\[ 200 + 40 + 5 = 245 \]

Final Answer: 532 - 287 = 245

Practice Exercise (Whole Numbers)

Question 1

Use partitioning to add: 468 + 379

Step 1: Partition both numbers

\[ 468 = 400 + 60 + 8 \] \[ 379 = 300 + 70 + 9 \]

Step 2: Add each place value

\[ 400 + 300 = 700 \] \[ 60 + 70 = 130 \] \[ 8 + 9 = 17 \]

Step 3: Combine the results

\[ 700 + 130 = 830 \] \[ 830 + 17 = 847 \]

Final Answer: 468 + 379 = 847

Question 2

Use partitioning to subtract: 725 - 368

Step 1: Partition both numbers

\[ 725 = 700 + 20 + 5 \] \[ 368 = 300 + 60 + 8 \]

Step 2: Subtract with regrouping

Hundreds: 700 - 300 = 400

Tens: Need to regroup because 20 < 60

\[ 400 = 300 + 100 \] \[ (100 + 20) - 60 = 120 - 60 = 60 \]

Ones: Need to regroup because 5 < 8

\[ 60 = 50 + 10 \] \[ (10 + 5) - 8 = 15 - 8 = 7 \]

Step 3: Combine the results

\[ 300 + 50 + 7 = 357 \]

Final Answer: 725 - 368 = 357

Adding & Subtracting Decimal Numbers

Decimal Addition Using Partitioning

Step-by-Step Method:

  1. Align numbers by decimal point
  2. Partition each number into whole and decimal parts
  3. Add whole numbers separately from decimal parts
  4. Combine the results
  5. Adjust for any regrouping needed across decimal point

Example: Add 4.37 + 2.86

Step 1: Partition both numbers

\[ 4.37 = 4 + 0.3 + 0.07 \] \[ 2.86 = 2 + 0.8 + 0.06 \]

Step 2: Add whole numbers and decimal parts separately

\[ 4 + 2 = 6 \] \[ 0.3 + 0.8 = 1.1 \] \[ 0.07 + 0.06 = 0.13 \]

Step 3: Combine the results

\[ 6 + 1.1 = 7.1 \] \[ 7.1 + 0.13 = 7.23 \]

Final Answer: 4.37 + 2.86 = 7.23

Decimal Subtraction Using Partitioning

Step-by-Step Method:

  1. Align numbers by decimal point
  2. Partition each number into whole and decimal parts
  3. Subtract whole numbers separately from decimal parts
  4. If needed, regroup across the decimal point
  5. Combine the results

Example: Subtract 5.2 - 3.75

Step 1: Write numbers with same decimal places

\[ 5.2 = 5.20 \] \[ 3.75 \text{ remains} \]

Step 2: Partition both numbers

\[ 5.20 = 5 + 0.2 + 0.00 \] \[ 3.75 = 3 + 0.7 + 0.05 \]

Step 3: Subtract with regrouping

Whole numbers: 5 - 3 = 2

Tenths: Need to regroup because 0.2 < 0.7

\[ 2 = 1 + 1 \] (take 1 whole as 10 tenths) \[ (1 + 0.2) - 0.7 = 1.2 - 0.7 = 0.5 \]

Hundredths: 0.00 - 0.05 → still need to regroup

\[ 0.5 = 0.4 + 0.10 \] (take 1 tenth as 10 hundredths) \[ 0.10 - 0.05 = 0.05 \]

Step 4: Combine the results

\[ 1 + 0.4 + 0.05 = 1.45 \]

Final Answer: 5.2 - 3.75 = 1.45

Practice Exercise (Decimal Numbers)

Question 1

Use partitioning to add: 6.45 + 3.78

Step 1: Partition both numbers

\[ 6.45 = 6 + 0.4 + 0.05 \] \[ 3.78 = 3 + 0.7 + 0.08 \]

Step 2: Add whole numbers and decimal parts

\[ 6 + 3 = 9 \] \[ 0.4 + 0.7 = 1.1 \] \[ 0.05 + 0.08 = 0.13 \]

Step 3: Combine the results

\[ 9 + 1.1 = 10.1 \] \[ 10.1 + 0.13 = 10.23 \]

Final Answer: 6.45 + 3.78 = 10.23

Question 2

Use partitioning to subtract: 7.3 - 4.86

Step 1: Write numbers with same decimal places

\[ 7.3 = 7.30 \] \[ 4.86 \text{ remains} \]

Step 2: Partition both numbers

\[ 7.30 = 7 + 0.3 + 0.00 \] \[ 4.86 = 4 + 0.8 + 0.06 \]

Step 3: Subtract with regrouping

Whole numbers: 7 - 4 = 3

Tenths: Need to regroup because 0.3 < 0.8

\[ 3 = 2 + 1 \] (take 1 whole as 10 tenths) \[ (1 + 0.3) - 0.8 = 1.3 - 0.8 = 0.5 \]

Hundredths: Need to regroup because 0.00 < 0.06

\[ 0.5 = 0.4 + 0.10 \] (take 1 tenth as 10 hundredths) \[ 0.10 - 0.06 = 0.04 \]

Step 4: Combine the results

\[ 2 + 0.4 + 0.04 = 2.44 \]

Final Answer: 7.3 - 4.86 = 2.44

Common Mistakes to Avoid

Frequent Errors

Misalignment of Place Values

Adding tens to ones or tenths to hundredths

Wrong:

\[ 34 + 25 = (30 + 4) + (20 + 5) \] \[ = (30 + 20) + (4 + 5) \ \text{✓}\]

But sometimes students do:

\[ (30 + 5) + (20 + 4) \ \text{✗}\]

Ignoring Decimal Alignment

Not lining up decimal points before adding/subtracting

Wrong:

\[ 4.5 + 3.26 = 4.5 \] \[ \underline{+\ 3.26} \] \[ 7.31 \text{✗ (should be 7.76)} \]

Incorrect Regrouping

Forgetting to adjust when moving across place values

Wrong:

\[ 52 - 37 \]

Trying to subtract 7 from 2 without regrouping

Omitting Place Values

Leaving out a place value during partitioning

Wrong:

\[ 405 = 400 + 5 \] (missing the 0 tens)

Pro Tips for Success

  • Always write numbers with the same number of decimal places when working with decimals
  • Use graph paper or draw lines to keep place values aligned
  • Double check your regrouping by verifying with standard algorithm
  • Estimate your answer first to check if your final result is reasonable
